## Branch type

> 브랜치 전략은 간략한 Git flow 전략을 사용합니다.
> 
- main (master) : 개발 서버 배포를 위한 브랜치
- #{issue number}/name : 기능 구현을 위한 브랜치

## Branch Naming

<aside>
💡 #{issue number}/{name}

</aside>

- 브랜치명은 반드시 **소문자로만 작성**해주시고, **케밥 케이스로 작성**합니다.
- name은 스프린트를 영어로 요약하여 작성해주세요.
